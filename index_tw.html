Agent 模式

AI 唔只係回答，仲可以自主執行

Agent 模式畀 AI 好似一位自主嘅開發者——連續調用多個工具，每步結果影響下一步決策，直到任務完成。

💬
Step 0
你發送需求
喺工作區對話中話畀 AI 知你想做咩——"修復登入頁樣式 bug"。系統自動對請求分類。
📦
Step 1
載入工作區上下文
自動讀取 README.md、package.json、Git 狀態、工作區記憶，AI 即刻了解你嘅專案。
🔍
Step 2
自主瀏覽與搜尋
AI 用 list_dir、read_file、search_files 等工具自己瀏覽專案結構、定位相關檔案。
🤔
Step 3
思考並調用工具
LLM 分析搜尋結果，決定下一步——可能讀取更多檔案、搜尋更多內容，或開始產生修改。
✏️
Step 4
產生修改並確認
AI 用 apply_patch 產生 unified diff——彈出確認對話框顯示修改摘要，你點擊「允許」後先寫入。
✅
Step 5
自動驗證與報告
寫入後自動執行 npm test / npm build 驗證。AI 最終回覆包含修改摘要、檔案列表、驗證結果。

14 內置工具

AI 嘅「雙手」，按風險分級管控

每個工具有明確嘅風險等級同審批策略——唯讀操作冇門檻，寫入操作需你確認。

📂 唯讀瀏覽
list_dir 🟢 safe 列出工作區目錄內容
read_file 🟢 safe 讀取工作區內文字檔案
search_files 🟢 safe 搜尋檔案路徑（ripgrep）
search_text 🟢 safe 搜尋檔案內容（正則+glob）
📄 唯讀檔案
txt_reader 🟢 safe 讀取用戶選擇嘅 .txt 檔案
markdown_reader 🟢 safe 讀取用戶選擇嘅 .md 檔案
git_status 🟢 safe 查看工作區 git 狀態
git_diff 🟢 safe 查看 git 工作區差異
✏️ 檔案建立 & 修改
create_file 🟡 medium 建立新檔案（唔可以覆蓋已有）
create_directory 🟡 medium 建立新目錄（需確認）
apply_patch 🟡 medium 應用 unified diff 修改檔案
⚡ 高風險操作
command_runner 🔴 high 執行授權目錄內嘅 shell 命令
update_memory 🔴 high 更新工作區持久化記憶
load_skill 🟢 safe 懶加載 Skill 完整 prompt

🛡️ 風險等級體系

🟢 safe 純唯讀操作，冇副作用——讀檔案、列目錄、搜尋、查看 git
🟡 medium 寫工作區內檔案，有限副作用——建立檔案/目錄、應用 patch（含檢查點備份）
🔴 high 執行命令、寫入記憶——每次操作彈出原生確認對話框
工作區記憶

AI 跨對話記住你嘅專案

工作區記憶係持久化嘅專案知識——AI 喺一次對話中學到嘅約定同偏好，下次對話仲可以記住。

🧠
資料庫記憶
AI 通過 update_memory 工具寫入資料庫。持續有效直到工作區被刪除。優先級最高。
📝
memory.md 檔案
工作區根目錄下 .ai-workbench/memory.md 檔案嘅內容。適合手動編輯或團隊共享，可提交到 git。
🔍
對話中查看
對話頁面頂部有「Memory」按鈕，點擊展開查看記憶來源、摘要（最多 280 字）、最後更新時間。

🤖 AI 更新記憶嘅流程

AI 判斷需要記住 → 調用 update_memory → 彈出確認對話框 → 你點擊「允許寫入」 → 記憶寫入資料庫

安全檢查：系統自動拒絕包含 API Key、Token、密碼等敏感資訊嘅記憶，亦拒絕睇落似對話記錄嘅內容。單條記憶最多 4000 字元。

🤔 技能點樣用？

  1. 自動匹配（懶加載）：訊息裡面包含關鍵詞時，AI 先收到 Skill 概要卡片，判斷需要時再通過 load_skill 載入完整 prompt——減少唔必要嘅上下文消耗。
  2. 手動選擇：輸入框旁邊嘅下拉菜單裡手動揀技能，直接使用完整 prompt，優先級最高。
  3. 單次生效：技能只影響當前呢條訊息，發送後自動返到預設狀態。
  4. 自訂 & 匯入：點擊「添加 Skill」建立專屬技能；亦可匯入包含 SKILL.md 嘅資料夾或 Markdown 檔案，自動解析 YAML 頭資訊。
安全與隱私

Command Runner 五層防護

命令執行器係風險最高嘅工具——因此我哋為佢設計咗最嚴密嘅五層安全防護，默認關閉，按需開啟。

1

手動啟用

喺設定頁面手動打開 command_runner 開關——默認關閉，唔開啟就冇法執行任何命令。

2

授權目錄

只能喺你明確添加嘅授權工作目錄內執行命令——其他路徑一律拒絕。

3

危險命令攔截

自動拒絕 14 種危險模式——rm -rf、del /s /q、sudo、format、shutdown 等，Shell 同 argv 雙模式覆蓋。

4

執行確認彈窗

每次執行前彈出原生確認對話框，顯示完整命令同工作目錄——你必須點擊「允許」先執行。

5

超時與取消

預設 60 秒超時（可調整到最大 300 秒），支援中途取消。輸出上限 20,000 字元。

🕵️

敏感資訊自動脫敏

工具日誌喺寫入本地資料庫前，自動將 API Key（sk-*）、Bearer Token、密碼等敏感資訊替換為 [REDACTED]。密鑰唔以明文落盤。檔案讀取只保存元數據同 500 字預覽。

🔐

API Key 加密保存

使用 Electron safeStorage 或 AES-256-GCM 加密。界面顯示 sk-****xxxx 格式掩碼。

💾

數據 100% 本地

所有對話保存喺本地 SQLite 資料庫（app.db）。冇中間伺服器，冇帳號系統。API Key 加密存儲喺本地。

Agent 模式係 AI Workbench 嘅高級工作方式。普通模式一問一答，你揀檔案畀 AI，AI 回答。

Agent 模式下，AI 自主多步驟執行：自己瀏覽專案結構 → 搜尋相關檔案 → 讀取內容 → 產生修改 → 你確認 → 應用修改 → 自動驗證。

兩種子模式：
・Ask（詢問）：傳統一問一答，適合簡單問答
・Craft（構建）：完整 Agent 執行，適合程式碼修改、專案分析

Agent 模式通過環境變數 AI_WORKBENCH_ENABLE_AGENT_MODE=1 啟用，需重啟應用。

Agent 運行有嚴格嘅安全限制，防止無限循環：

・牆鐘時間上限：10 分鐘
・總步驟數上限：12 步
・總工具調用上限：40 次
・同一工具重複調用上限：3 次（相同參數）
・連續無進展輪次：3 輪

高風險工具仲有重放防護——完全相同嘅參數再次調用會被拒絕，防止無效循環。apply_patch 每次寫入前自動建立 checkpoint（檔案備份）。
可能嘅原因：

1. 唔喺工作區對話中——update_memory 需要工作區上下文

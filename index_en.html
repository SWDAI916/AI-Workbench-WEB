Why AI Workbench

More than a chatbot — a true AI workbench

Regular AI chatbots only answer questions; AI Workbench lets AI autonomously execute tasks — reading files, searching code, modifying projects — all data stays local.

Common Pain Points 😰

Problems with Regular AI Tools

Most AI tools work one question at a time — you manually copy-paste file contents, and the AI can only give advice, never execute. Your data is stored on someone else's server.

  • One Q&A at a time, no autonomous multi-step execution
  • Must manually copy file contents to the AI
  • AI can only suggest, not modify your code
  • Chat history stored on the provider's servers
  • No project context — AI repeatedly misunderstands your intent
The Solution 😌

How AI Workbench Does It

AI Agent autonomously browses your project, searches code, modifies files, and runs commands — you state the goal, the AI handles understanding through execution. 100% local data.

  • Agent Mode for autonomous multi-step task execution
  • AI browses directories and searches files — no manual work
  • AI directly modifies code and auto-verifies changes
  • All conversations stored in local SQLite database
  • Workspace Memory lets AI remember project conventions across chats

🛡️ Risk Level System

🟢 safe Pure read-only operations, no side effects — read files, list directories, search, view git
🟡 medium Write to workspace files, limited side effects — create files/directories, apply patch (with checkpoint backup)
🔴 high Execute commands, write memory — a native confirmation dialog appears for each operation
Workspace Memory

AI remembers your project across conversations

Workspace Memory is persistent project knowledge — conventions and preferences the AI learns in one chat are remembered in the next.

🧠
Database Memory
AI writes to the database via the update_memory tool. Persists until the workspace is deleted. Highest priority.
📝
memory.md File
Contents of the .ai-workbench/memory.md file in the workspace root. Suitable for manual editing or team sharing — can be committed to git.
🔍
View in Chat
A "Memory" button at the top of the chat page expands to show memory sources, summaries (up to 280 characters), and last update time.

🤖 AI Memory Update Flow

AI decides to remember → Calls update_memory → Confirmation dialog appears → You click "Allow Write" → Memory written to database

Safety checks: The system auto-rejects memories containing API Keys, Tokens, passwords, or chat-like content. Max 4,000 characters per memory entry.

🤔 How Do Skills Work?

  1. Auto-match (Lazy Loading): When your message contains keywords, the AI first receives a skill summary card. If needed, it loads the full prompt via load_skill — reducing unnecessary context usage.
  2. Manual Selection: Pick a skill from the dropdown next to the input box — uses the full prompt directly. Highest priority.
  3. Single-Use: A skill only affects the current message and auto-resets to default after sending.
  4. Custom & Import: Click "Add Skill" to create your own; also supports importing folders containing SKILL.md or Markdown files, with automatic YAML front matter parsing.
Security & Privacy

Command Runner — 5-Layer Defense

The command runner is the highest-risk tool — that's why we built the strictest 5-layer security shield around it. Disabled by default; enable on demand.

1

Manual Activation

Toggle the command_runner switch on in Settings — disabled by default. No commands can execute until you enable it.

2

Authorized Directories

Commands can only run within directories you've explicitly authorized — all other paths are rejected.

3

Dangerous Command Blocking

Auto-rejects 14 dangerous patterns — rm -rf, del /s /q, sudo, format, shutdown, etc. — matched across both Shell and argv.

4

Execution Confirmation Dialog

A native confirmation dialog appears before each execution, showing the full command and working directory — you must click "Allow" to proceed.

5

Timeout & Cancel

Default 60-second timeout (configurable up to 300 seconds), with mid-execution cancellation. Output capped at 20,000 characters.

🕵️

Automatic Sensitive Data Redaction

Before tool logs are written to the local database, sensitive information such as API Keys (sk-*), Bearer Tokens, and passwords are automatically replaced with [REDACTED]. Keys are never stored in plaintext on disk. File reads only store metadata and a 500-character preview.

🔐

Encrypted API Key Storage

Uses Electron safeStorage or AES-256-GCM encryption. Displayed as masked format sk-****xxxx in the UI.

💾

100% Local Data

All conversations are stored in a local SQLite database (app.db). No intermediary server, no account system. API Keys are encrypted and stored locally.

Agent execution has strict safety limits to prevent infinite loops:

· Wall-clock time limit: 10 minutes
· Total step limit: 12 steps
· Total tool call limit: 40 calls
· Same-tool repeat call limit: 3 times (with identical parameters)
· Consecutive no-progress turns: 3 turns

High-risk tools also have replay protection — identical parameters called again will be rejected to prevent useless loops. apply_patch auto-creates a checkpoint (file backup) before each write.
Possible causes:

1. Not in a workspace conversation — update_memory requires workspace context
2. Content contains sensitive information — memory cannot include API Keys, Tokens, or passwords
3. Content looks like a chat transcript — the system rejects conversation-format content
4. Content too long — max 4,000 characters per memory entry
Checklist:

1. Is the command_runner switch turned on in Settings?
2. Has at least one authorized working directory been added?
3. Is the command's cwd within an authorized directory?
4. Did the command trigger a dangerous command block (14 patterns)?
5. Did you click "Allow" in the confirmation dialog?
Default storage locations:
· Windows: %APPDATA%/ai-workbench/app.db
· macOS: ~/Library/Application Support/ai-workbench/app.db
· Linux: ~/.config/ai-workbench/app.db

It's recommended to regularly back up the three files: app.db, app.db-wal, and app.db-shm. You can also change the storage location in Settings to point to a cloud-sync folder.

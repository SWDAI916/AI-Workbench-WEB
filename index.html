为什么选择 AI Workbench

不一样在哪里？

市面上 AI 工具很多，但大多数都把你的数据存在他们的服务器上。

常见痛点 😰

普通 AI 网页工具的问题

你和 AI 说的每一句话，都被存储在别人的服务器上。你的工作内容、私人想法、商业机密——都不在你的掌控之内。

  • 聊天记录存在服务商服务器
  • 服务商可能查看你的数据
  • 服务关闭后聊天记录消失
  • 涨价或政策变动你无法控制
解决方案 😌

AI Workbench 的做法

所有对话存在你电脑本地。AI Workbench 只是一个「中间桥梁」——它把你的消息发给 AI 服务商，把回复显示给你，全程不存储、不转发。

  • 所有对话保存在你的本地电脑
  • API Key 加密存储在本地
  • 没有中间服务器、没有账号系统
  • 你可以随时备份到 U 盘或网盘
三步上手

从安装到第一次对话

不需要任何技术背景，按照步骤来就行。

1

安装应用

下载对应你电脑系统的安装包——Windows 双击 .exe 按向导安装；macOS 打开 .dmg 拖入应用文件夹；Linux 直接运行 .AppImage。

2

配置一个 AI 模型

点击左侧「模型配置」，选择你用的 AI 服务商（如 DeepSeek、ChatGPT），填入你从服务商官网申请的 API Key，点击「测试连接」确认可用。

3

开始聊天

点击左侧「Chat」，在底部输入框输入内容，按 Enter 发送。AI 的回复会像打字一样逐字出现。点「+ 新建」可以随时开启新对话。

主要功能

不只是聊天

从日常对话到读取文件、生成报告，AI Workbench 都能帮你做到。

⚡
流式实时回复
AI 的回答逐字显示，不用干等一大段文字突然出现。生成中随时可以点「停止」。
💬
多对话管理
可以同时保留多个对话，按日期分组显示。支持重命名、删除，随时切换回来继续聊。
✏️
消息编辑与重新生成
发现问法不对？悬停在消息上点「编辑」修改后重新提交。对 AI 的回复不满意，点「重新生成」让它再答一次。
📁
按项目文件夹整理对话
将本地文件夹设为「工作区」，该工作区下的对话独立管理。处理多个项目时再也不会混淆上下文。
🔀
一键切换工作区
聊天页顶部显示当前工作区名称，点击即可快速切换。每个工作区只展示自己的对话，保持上下文清晰。
🛡️
文件操作以工作区为基准
读取文件、执行命令时默认从工作区根路径出发，减少路径混淆。工作区路径失效时会自动提示。
📄
读取你的本地文件
在聊天时选择一个 .txt 或 .md 文件，AI 会读取文件内容并据此回答你的问题——比如"帮我总结这份文档"。
📥
导出对话为 Markdown
单条 AI 回复或整个对话都可导出为 .md 文件，包含完整对话记录和工具调用摘要，方便存档或二次使用。
⚡
命令执行器（高级）
在授权的工作区文件夹内让 AI 执行本地命令。支持自然语言触发（如"查一下 Node 版本"），多层安全保护默认关闭，按需开启。
🎨
深色 / 浅色主题
支持跟随系统、手动浅色、手动深色。看起来护眼，用起来舒服。
📊
Markdown 渲染
AI 回复的代码块、表格、列表、标题都能正确显示，代码块还有一键复制按钮。
⚙️
灵活的界面布局
侧边栏和聊天区域都可以拖动调整宽度，完全按你的习惯来。宽度缩小后自动变成图标模式。

🤔 技能怎么用？

  1. 自动匹配：默认开启。你的消息里包含关键词（如"帮我写日报"），对应技能会自动激活。
  2. 手动选择：也可以在输入框旁边的下拉菜单里手动选择某个技能，强制使用它。
  3. 单次生效：技能只影响你当前这条消息，发送后自动回到默认状态，不会影响整个对话。
  4. 可以自定义：在「Skill」页面点击「添加 Skill」，用自然语言写下你想要 AI 扮演的角色和回答方式，就能创建专属技能。

安全与隐私

你的数据，你说了算

我们把「数据安全」放在第一位，而不是当作附加功能。

🔐

API Key 加密保存

你填入的 API Key 在保存前会经过加密处理，以密文形式存储在本地。应用界面显示的是 sk-****xxxx 格式的掩码，你自己也看不到原文，别人更看不到。

💾

聊天记录只在本地

所有对话都保存在你电脑上的数据库文件（app.db）里。AI Workbench 不会把你的聊天内容上传到任何服务器，也没有账号系统——本来就用不到。

📂

文件读取需要你主动授权

AI 不能自动读取你电脑上的任何文件。只有你在聊天页面点击「选择文件」按钮、主动选择了某个文件，AI 才能读取它。应用也不会保存文件的完整内容，只存简短预览。

⚠️

命令执行有五层保护

「命令执行」功能默认是关闭的，需要你在设置里手动开启，并指定允许操作的文件夹。每次执行命令前还会弹出确认窗口，危险操作（如删除文件）会被自动拦截。

🕵️

敏感信息自动脱敏

工具执行产生的日志在写入本地数据库之前，会自动将 API Key、Token、密码等敏感信息替换为 [REDACTED]，确保密钥不以明文形式留存在任何记录中。

工作区就是把你电脑上的一个文件夹和你的 AI 对话绑定在一起。创建后：

・对话列表只展示该工作区的对话，不同项目互不干扰
・读取文件或执行命令时默认从该文件夹出发，路径更准确
・可以为每个项目单独创建工作区，切换时一键搞定

如果你只是随手聊，不绑定工作区也完全没问题，所有对话照常保存。

自动匹配依赖「关键词」触发。检查以下几点：

1. 你的消息里是否包含了触发词？（比如"日报生成"需要消息里含有「日报」）
2. 对应的技能是否已经打开（在「Skill」页面确认开关是开启状态）？
3. 输入框旁的选择器是否处于「自动匹配」模式？

如果自动匹配不稳定，可以直接在选择器里手动选技能，优先级最高。
默认存储位置：
・Windows：%APPDATA%/ai-workbench/app.db
・macOS：~/Library/Application Support/ai-workbench/app.db

你可以在「设置」页面看到具体路径。建议定期把这个 .db 文件复制到 U 盘或网盘作备份，防止电脑损坏时丢失。也可以在设置里更改存储位置，直接指向你的网盘同步文件夹。
文件读取有几个限制：

1. 必须通过聊天页面的「选择文件」按钮来选择文件，不能手动输入路径。
2. 只支持 .txt、.md、.markdown 三种格式。
3. 文件大小不能超过 5MB。
4. 文件授权信息在应用关闭后会清除，重新打开应用后需要重新选择文件。
在 Windows 上卸载时，安装程序会询问你是否同时删除本地数据（聊天记录、模型配置、密钥等）。你可以选择保留，之后重新安装后数据依然在。

如果你担心数据安全，建议卸载前先手动找到 app.db 文件备份，或在设置里点「清理会话历史」再卸载。

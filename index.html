为什么选择 AI Workbench

不只是聊天工具，是真正的 AI 工作台

普通 AI 只能回答问题；AI Workbench 让 AI 自主执行任务——读文件、搜代码、修改项目，全程数据留在本地。

常见痛点 😰

普通 AI 工具的问题

大多数 AI 工具只能一问一答，你需要手动复制粘贴文件内容，然后 AI 只能给你建议，不能帮你执行。数据还被存在别人的服务器上。

  • 只能一问一答，不能自主执行多步骤
  • 需要手动复制文件内容给 AI
  • AI 只能建议，不能帮你修改代码
  • 聊天记录存在服务商服务器上
  • 没有项目上下文，AI 反复误解你的意图
解决方案 😌

AI Workbench 的做法

AI Agent 自主浏览项目、搜索代码、修改文件、执行命令——你只说需求，AI 从理解到执行全自动。所有数据 100% 保存在本地。

  • Agent 模式自主多步骤执行任务
  • AI 自己浏览目录、搜索文件，无需手动
  • AI 可以直接修改代码并自动验证
  • 所有对话保存在本地 SQLite 数据库
  • 工作区记忆让 AI 跨对话记住项目约定
三步上手

从安装到 Agent 执行

不需要任何技术背景，按照步骤来就行。

1

安装并配置

下载安装包，选择你用的 AI 服务商（DeepSeek、ChatGPT、Claude 等），填入 API Key，点击「测试连接」确认可用。

2

创建工作区

将你的项目文件夹设为「工作区」。AI 会自动加载项目上下文——README、package.json、Git 状态等，无需手动说明。

3

切换到 Craft 模式

在 Craft 模式下告诉 AI 你想做什么——"修复登录页的样式 bug"。AI 自主浏览项目、定位问题、生成修改、自动验证，全程透明可控。

Agent 模式

AI 不只是回答，还能自主执行

Agent 模式让 AI 像一位自主的开发者——连续调用多个工具，每步结果影响下一步决策，直到任务完成。

💬
Step 0
你发送需求
在工作区对话中告诉 AI 你想做什么——"修复登录页样式 bug"。系统自动对请求分类。
📦
Step 1
加载工作区上下文
自动读取 README.md、package.json、Git 状态、工作区记忆，AI 立刻了解你的项目。
🔍
Step 2
自主浏览与搜索
AI 用 list_dir、read_file、search_files 等工具自己浏览项目结构、定位相关文件。
🤔
Step 3
思考并调用工具
LLM 分析搜索结果，决定下一步——可能读取更多文件、搜索更多内容，或开始生成修改。
✏️
Step 4
生成修改并确认
AI 用 apply_patch 生成 unified diff——弹出确认对话框显示修改摘要，你点击「允许」后才写入。
✅
Step 5
自动验证与报告
写入后自动运行 npm test / npm build 验证。AI 最终回复包含修改摘要、文件列表、验证结果。

14 内置工具

AI 的「双手」，按风险分级管控

每个工具有明确的风险等级和审批策略——只读操作无门槛，写入操作需你确认。

📂 只读浏览
list_dir 🟢 safe 列出工作区目录内容
read_file 🟢 safe 读取工作区内文本文件
search_files 🟢 safe 搜索文件路径（ripgrep）
search_text 🟢 safe 搜索文件内容（正则+glob）
📄 只读文件
txt_reader 🟢 safe 读取用户选择的 .txt 文件
markdown_reader 🟢 safe 读取用户选择的 .md 文件
git_status 🟢 safe 查看工作区 git 状态
git_diff 🟢 safe 查看 git 工作区差异
✏️ 文件创建 & 修改
create_file 🟡 medium 创建新文件（不可覆盖已有）
create_directory 🟡 medium 创建新目录（需确认）
apply_patch 🟡 medium 应用 unified diff 修改文件
⚡ 高风险操作
command_runner 🔴 high 执行授权目录内的 shell 命令
update_memory 🔴 high 更新工作区持久化记忆
load_skill 🟢 safe 懒加载 Skill 完整 prompt

🛡️ 风险等级体系

🟢 safe 纯只读操作，无副作用——读文件、列目录、搜索、查看 git
🟡 medium 写工作区内文件，有限副作用——创建文件/目录、应用 patch（含检查点备份）
🔴 high 执行命令、写入记忆——每次操作弹出原生确认对话框
工作区记忆

AI 跨对话记住你的项目

工作区记忆是持久化的项目知识——AI 在一次对话中学到的约定和偏好，下次对话还能记住。

🧠
数据库记忆
AI 通过 update_memory 工具写入数据库。持续有效直到工作区被删除。优先级最高。
📝
memory.md 文件
工作区根目录下 .ai-workbench/memory.md 文件的内容。适合手动编辑或团队共享，可提交到 git。
🔍
聊天中查看
聊天页面顶部有「Memory」按钮，点击展开查看记忆来源、摘要（最多 280 字）、最后更新时间。

🤖 AI 更新记忆的流程

AI 判断需要记住 → 调用 update_memory → 弹出确认对话框 → 你点击「允许写入」 → 记忆写入数据库

安全检查：系统自动拒绝包含 API Key、Token、密码等敏感信息的记忆，也拒绝看起来像聊天记录的内容。单条记忆最多 4000 字符。

技能（Skills）

让 AI 按你想要的方式回答

技能是预设的 Prompt 模板——激活后 AI 按特定风格和格式处理你的输入。支持懒加载、自定义、导入导出。

📝
文本总结
把一段长文本或一篇文档提炼成简洁的摘要，抓住核心要点。
总结摘要提炼概括
📋
日报生成
把你零散的工作记录整理成格式规范的日报，几秒钟搞定每天的汇报。
日报工作汇报今日工作
✍️
公众号改写
把普通文字改写成适合微信公众号发布的文章风格，语言更流畅，更有感染力。
公众号改写推文

🤔 技能怎么用？

  1. 自动匹配（懒加载）：消息里包含关键词时，AI 先收到 Skill 概要卡片，判断需要时再通过 load_skill 加载完整 prompt——减少不必要的上下文消耗。
  2. 手动选择：输入框旁的下拉菜单里手动选技能，直接使用完整 prompt，优先级最高。
  3. 单次生效：技能只影响当前这条消息，发送后自动回到默认状态。
  4. 自定义 & 导入：点击「添加 Skill」创建专属技能；也可导入包含 SKILL.md 的文件夹或 Markdown 文件，自动解析 YAML 头信息。
安全与隐私

Command Runner 五层防护

命令执行器是风险最高的工具——因此我们为它设计了最严密的五层安全防护，默认关闭，按需开启。

1

手动启用

在设置页面手动打开 command_runner 开关——默认关闭，不开启就无法执行任何命令。

2

授权目录

只能在你明确添加的授权工作目录内执行命令——其他路径一律拒绝。

3

危险命令拦截

自动拒绝 14 种危险模式——rm -rf、del /s /q、sudo、format、shutdown 等，Shell 和 argv 双模式覆盖。

4

执行确认弹窗

每次执行前弹出原生确认对话框，显示完整命令和工作目录——你必须点击「允许」才执行。

5

超时与取消

默认 60 秒超时（可调整到最大 300 秒），支持中途取消。输出上限 20,000 字符。

🕵️

敏感信息自动脱敏

工具日志在写入本地数据库前，自动将 API Key（sk-*）、Bearer Token、密码等敏感信息替换为 [REDACTED]。密钥不以明文落盘。文件读取只保存元数据和 500 字预览。

🔐

API Key 加密保存

使用 Electron safeStorage 或 AES-256-GCM 加密。界面显示 sk-****xxxx 格式掩码。

💾

数据 100% 本地

所有对话保存在本地 SQLite 数据库（app.db）。没有中间服务器，没有账号系统。API Key 加密存储在本地。

Agent 模式是 AI Workbench 的高级工作方式。普通模式一问一答，你选文件给 AI，AI 回答。

Agent 模式下，AI 自主多步骤执行：自己浏览项目结构 → 搜索相关文件 → 读取内容 → 生成修改 → 你确认 → 应用修改 → 自动验证。

两种子模式：
・Ask（询问）：传统一问一答，适合简单问答
・Craft（构建）：完整 Agent 执行，适合代码修改、项目分析

Agent 模式通过环境变量 AI_WORKBENCH_ENABLE_AGENT_MODE=1 启用，需重启应用。

Agent 运行有严格的安全限制，防止无限循环：

・墙钟时间上限：10 分钟
・总步骤数上限：12 步
・总工具调用上限：40 次
・同一工具重复调用上限：3 次（相同参数）
・连续无进展轮次：3 轮

高风险工具还有重放防护——完全相同的参数再次调用会被拒绝，防止无效循环。apply_patch 每次写入前自动创建 checkpoint（文件备份）。
可能的原因：

1. 不在工作区对话中——update_memory 需要工作区上下文
2. 内容包含敏感信息——记忆不能包含 API Key、Token、密码
3. 内容看起来像聊天记录——系统拒绝保存对话格式的内容
4. 内容过长——单条记忆不超过 4000 字符
检查清单：

1. 设置页面中是否已打开 command_runner 开关？
2. 是否已添加至少一个授权工作目录？
3. 命令的 cwd 是否在授权目录范围内？
4. 命令是否触发了危险命令拦截（14 种模式）？
5. 是否在确认弹窗中点击了「允许」？

・Windows：%APPDATA%/ai-workbench/app.db
・macOS：~/Library/Application Support/ai-workbench/app.db
・Linux：~/.config/ai-workbench/app.db

建议定期备份 app.db、app.db-wal、app.db-shm 三个文件。也可以在设置里更改存储位置，指向网盘同步文件夹。
